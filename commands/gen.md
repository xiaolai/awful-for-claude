# /awful:gen

Generate workflow files from the awful blueprint. Reads `workflows/BLUEPRINT.md`, determines output format, and produces `.github/workflows/` files ready to commit.

## Steps

### Step 1: Read Blueprint

Read `workflows/BLUEPRINT.md`. If it doesn't exist:
```
No blueprint found at workflows/BLUEPRINT.md.
Run /awful:design first to create a blueprint.
```

Parse the YAML frontmatter. Validate:
- `agents:` array is present and non-empty
- Each agent has `name` and `role`
- `routing:` array has at least one entry
- Each routing entry references a valid agent name

Report any validation errors clearly before proceeding.

### Step 2: Determine Format

Run `commands/shared/format.md` to get the output format. Display the selection:

```
Output format: {gh-aw markdown | GitHub Actions YAML}
```

### Step 3: Create Output Directory

```bash
mkdir -p .github/workflows
```

For GitHub Actions YAML format with separate prompt files, also:
```bash
mkdir -p .github/workflows/agents
```

### Step 4: Dispatch Generator

Dispatch the **generator agent** (`agents/generator.md`) with:
- Blueprint content (YAML frontmatter + markdown body)
- Selected output format
- List of available templates from `templates/`
- Instruction to generate all workflow files

The generator will:
1. Process each agent from the blueprint
2. Check if the agent role matches a built-in template
3. Generate the workflow file in the selected format
4. Create the labels file

### Step 5: Post-Generation Summary

After generation completes, show:

```
Generated {n} workflow file(s) ({format}):

{for each file:}
  .github/workflows/{agent-name}.{md|yml}
    Event: {trigger event}
    Permissions: {list}
    Loop prevention: bot-actor-guard{, fork-PR-guard if PR-triggered}

Also created:
  .github/labels.yml — {n} labels

{If gh-aw format:}
Next steps:
  1. Review the generated files
  2. Commit and push
  3. Apply labels: gh label import .github/labels.yml
  4. Compile workflows: gh aw compile .github/workflows/*.md

{If Actions YAML format:}
Next steps:
  1. Review the generated files
  2. Add ANTHROPIC_API_KEY as a repo secret: gh secret set ANTHROPIC_API_KEY
  3. Commit and push
  4. Apply labels: gh label import .github/labels.yml
```

### Overwrite Behavior

If workflow files already exist:
- Check if they were generated by awful (look for `# Generated by awful` comment at top)
- If yes: overwrite silently
- If no (hand-written files): ask before overwriting

### Regeneration

If the user runs `/awful:gen` again after updating the blueprint:
- Read the updated blueprint
- Regenerate all files
- Report which files changed vs stayed the same
